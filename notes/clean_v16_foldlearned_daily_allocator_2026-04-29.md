# Clean V16 Fold-Learned Daily Allocator

Run directory: `logs\20260429_140328_clean_v16_foldlearned_daily_allocator`

## Boundary

This is **clean-input research**. It rebuilds V10 from raw/train inputs and uses train-only rolling folds to learn daily share corrections. It does not read test targets, sample submission, previous submissions, or quarantine files.

## Hypothesis

V14 hand-tuned month-day/day-of-week allocation reached a plateau. V16 learns a correction to the train-derived daily profile from historical rolling folds, including optional month-boundary features, while preserving each month total.

## Public Result

- `submission_cleanv16_v10_boundary_both_c500_a300.csv`: `667551.14004`
- `submission_cleanv16_v10_boundary_both_c750_a300.csv`: `668194.03025`
- This is the current clean/public-guided best, improving over `submission_cleanv14_v10_all_mddow_both_a300.csv = 667774.94896`.
- Interpretation: learned daily allocation is a real signal, but stronger correction over-shapes public. Keep `correction_alpha` near `0.50`; do not escalate to `c1000_a300` unless doing postmortem mapping.
