## Next Test Routes After `tail_ramp40 = 911k`
- Route A: different recursive regime model, not post-process calibration.
  - The next serious branch should change the training/regime structure itself, not adjust public predictions after the fact.
  - Candidate tests: `train_window_days` around recent windows, `piecewise_exp` weighting, and explicit 2021-2022 regime-only donors.
- Route B: feature simplification and noise pruning.
  - The ablation report says bigger is not better: `full_model` has `971` features and is worse offline than smaller sets.
  - The current useful branch uses `curated_promo_cogs` with about `190` features.
  - Next tests should stay around compact feature sets, not expand to 500-971 feature monsters unless there is a specific reason.
- Route C: recency anchor preservation.
  - `recencyexp20` is now the best known public anchor.
  - New submissions should keep it as the baseline unless a candidate beats it offline and has a genuinely different structural reason.
- Route D: stop broad revenue shrink / seasonal re-anchor.
  - `tail_ramp40` proved the seasonal pull was too destructive on public.
  - The next calibration tests, if any, must be narrow and segment-specific, not whole-horizon smooth shrink.
